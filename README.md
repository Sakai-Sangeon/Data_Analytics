# Data_Analytics

Project: Customer Churn Prediction

Goal: Predict which customers are likely to churn based on behavioural and demographic variables.

Approach:


Generated a synthetic churn dataset using AI assistance to simulate realistic customer behaviour patterns
Cleaned and structured the data using Python (Pandas)
Ran correlation analysis to identify variables most associated with churn
Built a predictive model and visualised outputs in Power BI


Tools: Python, Pandas, Power BI

Dashboard Preview:

Tracked core churn KPIs including total customers (20,000), current churn rate (8.0%), average churn probability (15.0%), and estimated revenue at risk ($560.4K)
Analysed correlation between dataset features and churn rate, identifying support ticket volume and last interaction days as the strongest risk drivers, and CSAT score as the most protective factor
Quantified expected revenue loss by risk segment: High ($344.9K), Medium ($122.5K), and Low ($93.0K), based on mean churn probability per group
Modelled the financial impact of retention actions, showing that a 10% improvement in CSAT recovers $180.4K in revenue, four times the impact of a 10% reduction in support ticket volume ($45.9K)
Built as an actionable dashboard with interactive filters across contract type, risk segment, subscription tier, and action status, enabling business-level analysis across different customer segments
